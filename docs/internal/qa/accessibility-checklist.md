# Accessibility Checklist

- [ ] semantic landmarks and heading order are correct
- [ ] links navigate and buttons perform commands
- [ ] every form input has a visible label and associated error
- [ ] validation summary receives useful focus/status behavior
- [ ] loading/mutation/error updates are announced appropriately
- [ ] all workflows work with keyboard only
- [ ] focus is visible and restored after navigation/mutation
- [ ] status is not communicated by color alone
- [ ] route title and active navigation are meaningful
- [ ] content works at 200% zoom and mobile width
- [ ] lazy loading does not leave an inaccessible blank screen
