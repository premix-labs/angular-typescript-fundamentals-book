# รู้จัก Angular 22, SPA และ Framework Mental Model

> สถานะบท: โครงร่างสำหรับพัฒนาต้นฉบับ

## เป้าหมาย

- อธิบายความต่างระหว่าง browser page, SPA และ Angular application
- อ่าน component tree และ data flow ของ Product Catalog
- แยกหน้าที่ของ component, template, service, router และ dependency injection
- เข้าใจภาพรวมว่า Angular 22 อัปเดตหน้าจอแบบ zoneless เมื่อ framework ได้รับ notification จาก Signals, template listeners หรือ async bindings
- รู้ว่าหัวข้อใดยังไม่ควรนำมาใช้ก่อนถึงบทที่สอน

## Milestone

วาด application map ของ list, detail, create, edit และ not-found โดยยังไม่สร้างโปรเจกต์

## Checkpoint

ผู้เรียนอธิบายได้ว่าทำไม component ไม่ควรเรียกทุก responsibility เอง และข้อมูลแต่ละชนิดควรมีเจ้าของที่ชัดเจน
